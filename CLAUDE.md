# Project Notes for Claude

## Random Number Generation

Random operations are assumed to be expensive. Where multiple random bytes of known length are needed in short succession (e.g., a 16-byte vec and possibly a single `u8`), a 17-byte stack-allocated buffer should be used, with copies taken from this buffer rather than making separate RNG calls.

## Code Generation

- **Do not manually edit generated files.** All files in `radius/src/dict/` with the header `// Code generated by machine generator; DO NOT EDIT.` are generated from `radius/dicts/` by the `code-generator` crate.
- Trigger regeneration by touching `radius/build.rs` then running `cargo build --package radius`. The `build.rs` calls `code_generator::generate(&out_dir, &dict_file_path_refs)` on every dict file in `radius/dicts/`.
- One `.rs` file is emitted per dictionary file. The generator also runs `rustfmt --edition=2021` on its output internally.
- **Standard RFC attributes** (no `BEGIN-VENDOR` block) get: an `{ATTR}_TYPE: AVPType` constant, plus `add_*`, `delete_*`, `lookup_*`, and `lookup_all_*` helpers on `Packet`.
- **Vendor-Specific Attributes (VSA)** inside `BEGIN-VENDOR`/`END-VENDOR` blocks get: an `{ATTR}_VENDOR_TYPE: u8` constant with the vendor sub-type, plus `add_*`, `delete_*`, `lookup_*`, and `lookup_all_*` helpers that use `AVP::from_vsa` / `Packet::lookup_vsa` / `Packet::delete_vsa` to encode/decode properly as RADIUS type-26 Vendor-Specific AVPs.
- Attribute names are transformed via the `inflector` crate: `snake_case` for methods, `PascalCase` for types, `SCREAMING_SNAKE_CASE` for value constants.
- Sub-attributes with dotted type IDs (e.g. `153.2`) are silently skipped.
- Duplicate attribute type numbers within the same vendor scope are silently skipped.

## Crate Structure

- **`radius`** — the main library crate; exposes `client`, `server`, and `core`.
- **`code-generator`** — internal binary; not published; reads dict files and writes source.
- **`examples`** — separate crate with `client.rs` and `server.rs` example binaries.
- Workspace resolver is `"2"`; MSRV is **Rust 1.75**.

## Crypto Backends

Exactly one of three mutually exclusive features must be active (enforced via `compile_error!` in `lib.rs`):

- **`aws-lc`** (default) — uses `aws-lc-sys` FFI directly; `RAND_bytes` and `MD5` from AWS-LC.
- **`openssl`** — uses the `openssl` and `rand` crates.
- **`rust-crypto`** — pure-Rust backend using RustCrypto crates (`md-5`, `md4`, `sha1`, `hmac`, `des`) + `rand`.

The unified API in `radius/src/core/crypto.rs` exposes: `md5(data) -> [u8; 16]`, `random_bytes(n) -> Vec<u8>`, `fill_random(buf: &mut [u8])`.

To test all backends: `make check_all` (also `make check_openssl`, `make check_rust_crypto`).

## Key Abstractions

- **`Packet`** — core struct (`code`, `identifier`, `authenticator`, `secret`, `attributes`). Max wire size 4096 bytes. `Packet::new` picks a random identifier; use `Packet::new_with_identifier` when you need a specific one. `Packet::decode` parses wire bytes.
- **`AVP`** — `{ typ: AVPType (u8), value: Bytes }`. Constructors/decoders for every RADIUS value type including `from_user_password` and `from_tunnel_password` (encrypted variants).
- **`Request`** — wraps an incoming `Packet` with `local_addr` and `remote_addr`.
- **`Client`** — UDP; configurable connection/socket timeouts; no built-in retransmission.
- **`Server`** — async Tokio UDP server; requires a `RequestHandler` and `SecretProvider`; deduplicates in-flight requests via `Arc<RwLock<HashSet<RequestKey>>>`.

## Conventions

- All error types use `thiserror` and implement `PartialEq` + `Debug`.
- `Bytes`/`BytesMut` (from the `bytes` crate) are used for all binary payloads throughout.
- `#[must_use]` is applied to all getters and lookup functions; the `clippy::double_must_use` lint is suppressed in generated files because lookups return `Option<Result<T, AVPError>>`.
- There are no public re-exports at the crate root — callers import `radius::core::rfc2865`, `radius::client::Client`, etc. directly.
